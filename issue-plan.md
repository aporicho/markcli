# Issue 修复顺序

综合依赖关系、难度和价值，推荐以下修复顺序：

---

### 第一梯队：基础 MCP 工具（后续 issue 的前置依赖）

| 顺序 | Issue | 理由 |
|------|-------|------|
| 1 | **#2 clear_annotations** | 最简单的新 MCP 工具，#8 依赖它，且 #5 降级模式也涉及它 |
| 2 | **#4 resolve_annotation** | #8 也依赖它，给批注加 `resolved` 状态，改动明确 |
| 3 | **#8 bug: 修改后未清理批注** | 工具层（#2 #4）就绪后，这个本质上是 **CLAUDE.md 提示词更新**，几分钟搞定 |

### 第二梯队：核心体验 bug

| 顺序 | Issue | 理由 |
|------|-------|------|
| 4 | **#10 文件修改后视图不同步** | 核心使用流程的阻断性 bug，已有 `useFileWatcher` 基础，排查修复 |
| 5 | **#9 输入面板高度不自适应** | Linux 下的 UI bug，可能是 Ink 跨平台问题，需要调试 |

### 第三梯队：MCP 能力增强

| 顺序 | Issue | 理由 |
|------|-------|------|
| 6 | **#5 无 TUI 降级模式** | 让所有读写类 MCP 工具脱离 TUI 运行，提升整体可用性 |
| 7 | **#3 jump_to_annotation** | 需要 TUI 滚动逻辑，中等复杂度 |

### 第四梯队：视觉增强（可选做）

| 顺序 | Issue | 理由 |
|------|-------|------|
| 8 | **#7 交替配色 + 序号徽章** | 依赖 #6 的主题系统，��也可先用硬编码颜色独立做 |
| 9 | **#6 主题系统** | 工作量最大的 feature，纯视觉增强，不阻塞功能 |
| 10 | **#1 install.sh 优化** | 用户体验改进，优先级最低，不影响核心功能 |

---

**核心逻辑**：先补齐 MCP 工具链（#2→#4→#8），再修核心 bug（#10→#9），然后增强降级能力（#5→#3），最后做视觉和安装体验。
